# 機能ガイド：コマンド・エージェント・スキル早見表

本プロジェクトは、Antigravity（Gemini）専用に適合化された株式調査（エクイティリサーチ）ツールセットです。
すべての財務モデリング、決算分析、セクター調査の成果物は **Google Workspace（Google スプレッドシート）** での閲覧・編集に完全に対応しています。

---

## 1. Google Workspace（Google スプレッドシート）への適合仕様

本プロジェクトで生成される財務モデルは、以下の互換仕様に従って出力されます。

- **ファイル形式**: すべての財務モデルは Python の `openpyxl` ライブラリを使用して `.xlsx` 形式のファイルとして生成されます。成果物ファイルを Google ドライブにドラッグ＆ドロップするだけで、Google スプレッドシートとしてネイティブにインポートおよび編集が可能です。
- **ライブ連携アドインの廃止**: ローカルの Excel アプリケーションや Office JS API (Excel Liveアドイン) への依存を完全に排除し、完全に headless なファイル作成に特化しています。
- **数式の互換性**: Google スプレッドシートと互換性のない Excel 独自の「データテーブル」機能やアドイン関数は使用せず、すべて標準的なスプレッドシート数式（`INDEX`, `MATCH`, `OFFSET`, `IF`, `MEDIAN`, `QUARTILE` など）のみで動的に構築されます。
- **循環参照（反復計算）の自動有効化**: 3-statementモデル等の金利計算における循環参照を解消するため、openpyxl で保存する段階でファイル内に「反復計算（Iterative Calculation）を有効化」する設定メタデータを埋め込みます。これにより、Google スプレッドシートでインポートして開いた時点で、ユーザーが手動で設定を変更することなく、最初から自動的に再計算が行われます。

## 2. スラッシュコマンド（ワークフロー）

チャット入力欄に `/` を入力することで、以下の明示的なワークフロー（スラッシュコマンド）を起動できます。引数（銘柄や条件）を省略した場合は、エージェントが対話的に必要な情報を質問します。

| コマンド | 引数 | 実行すると何ができるか | 使用する主なスキル |
|---|---|---|---|
| `/earnings` | 銘柄, 四半期 | 四半期決算を分析し、**決算アップデートレポート**の草案を作成。実績とコンセンサス予想の beat/miss 定量化、セグメント別分析、ガイダンスの修正、投資仮説への影響を記述。 | `earnings-analysis` |
| `/earnings-preview` | 銘柄 | **決算発表前**のプレビュー。コンセンサス予想、注目すべきKPI、強気/標準/弱気の業績シナリオを整理。 | `earnings-preview` |
| `/initiate` | 銘柄 | **新規カバレッジ開始レポート**を作成。会社概要、ビジネスモデル、業界トレンド、強み、バリュエーション、リスク分析を網領する最も重厚なレポート構築ワークフロー。 | `initiating-coverage` |
| `/model-update` | 銘柄 | 既存の財務モデルに新しい決算実績、ガイダンス、および改訂前提を反映して更新。 | `model-update` |
| `/morning-note` | なし | 担当ユニバース横断で、**朝会用の簡潔なノート**（寄り付き前想定）を作成。オーバーナイトの動き、決算速報、トレードアイデアを集約。 | `morning-note` |
| `/screen` | 定量条件 | 定量スクリーンやテーマ的スイープにより、**新規の投資アイデアをスクリーニング**。 | `idea-generation` |
| `/sector` | セクター/業界 | **セクター概観レポート**を作成。市場規模、競争構造、主要プレイヤー、今後の投資の含意を記述。 | `sector-overview` |
| `/thesis` | 銘柄 | **投資仮説（thesis）の新規作成・更新**。新しいデータポイントを反映して仮説を管理。 | `thesis-tracker` |
| `/catalysts` | 期間 | カバレッジ銘柄横断の**カタリスト・カレンダー**（決算日、法規制、製品発表など）を構築。 | `catalyst-calendar` |
| `/comps` | 銘柄 | **類似企業比較分析（trading comps）**を構築。4〜6社のピア銘柄選定、営業指標（売上/成長率/粗利/EBITDA）とバリュエーション倍率（EV/Rev, EV/EBITDA, P/E）の比較、統計サマリー（最大/75%/中央値/25%/最小）を数式ベースの `.xlsx` で出力。 | `comps-analysis` |
| `/dcf` | 銘柄 | **DCFバリュエーションモデル**を構築。自動的に `/comps` を回してピア倍率を取得し、それをターミナル倍率（exit multiple）の前提に使用。WACC（CAPM）、Bear/Base/Bullの3ケース、感応度表（5x5グリッド）を含むモデル（`.xlsx`）を生成。 | `comps-analysis` → `dcf-model` |
| `/lbo` | 企業, 条件 | **LBOモデル**（レバレッジドバイアウト）を構築。Sources & Uses、デットスケジュール、リターンウォーターフォール、IRR/MOIC感応度を網羅したモデル（`.xlsx`）を生成。 | `lbo-model` |
| `/3-statement-model`| なし（対話） | **3表連動モデル**（PL/BS/CFが統合され、運転資本やデットスケジュールが連動した財務モデル）をテンプレートに基づいて作成。 | `3-statement-model` |
| `/competitive-analysis`| 企業/業界 | **競合環境分析**。主要プレイヤーのポジショニング、競争力の源泉、最近の戦略的動向を分析。 | `competitive-analysis` |
| `/debug-model` | モデルのパス | 既存の `.xlsx` モデルを**監査・デバッグ**。壊れた数式（#REF!等）、バランスシート不一致、数式セルへの数値ハードコード、誤った循環参照、ロジックの不整合などを検出して修正案を提示。 | `audit-xls` |
| `/ppt-template` | テンプレートパス | 既存の PowerPoint テンプレート（`.pptx`/`.potx`）を解析し、自社ブランド様式に準拠したスライド生成用カスタムスキルを自動構築。 | `ppt-template-creator` |

## 3. エージェント（ワークフロー一括実行）

サブエージェントとしてバックグラウンドで起動し、複数のステップにまたがる複雑なタスクを人間の承認を挟みながら自動実行します。

| エージェント | 起動方法 | 役割と出力 | 主な使用スキル |
|---|---|---|---|
| **market-researcher** | `/agent-market-researcher` | セクターや特定のテーマに関する**一次調査**を一括実行。①業界概観（規模/成長/構造/ドライバー/why-now）→②競合マッピング→③ピアcomps分析→④テーマを体現する3〜5銘柄の推奨ショートリスト→⑤リサーチノートの作成。comps後とノート完成後にレビューのため停止します。 | `sector-overview`<br>`competitive-analysis`<br>`comps-analysis`<br>`idea-generation`<br>`pptx-author` |
| **earnings-reviewer** | `/agent-earnings-reviewer` | カバレッジ銘柄が決算を発表した際の**決算処理を端から端まで自動実行**。①実績データ・決算資料・コール文字起こしの取得→②ガイダンスや質疑応答のトーン分析→③カバレッジ財務モデルの更新→④モデル監査（audit-xls）→⑤決算ノート草案および実績vs予想の差異テーブルの作成。 | `earnings-analysis`<br>`model-update`<br>`audit-xls`<br>`morning-note`<br>`earnings-preview` |
| **model-builder** | `/agent-model-builder` | 銘柄と前提条件から**完全リンクされた財務モデル（.xlsx）をゼロから構築**。DCF/LBO/3表/compsのいずれか。①実績財務データの取得→②数式モデルの構築（計算セルへのハードコード禁止、反復計算自動有効化、入力＝青/数式＝黒）→③モデルの監査→④感応度表（感度分析）の追加。 | `dcf-model`<br>`lbo-model`<br>`3-statement-model`<br>`comps-analysis`<br>`audit-xls` |

## 4. カスタムスキル（`.agents/skills/`）

Antigravity のバックグラウンドで自動検出され、必要な文脈において自動的または明示的にロードされて実行される実作業のロジックです。

- **`dcf-model` / `lbo-model` / `3-statement-model` / `comps-analysis`**: 各種財務モデルの構築・計算ルール。
- **`japan-equity-overrides` (新設)**: 日本株の財務分析に必須のローカライズ共通ルール（有報/決算短信の優先度、通貨単位 円/百万円、3月期決算カレンダー、実効税率約30%、10年JGBリスクフリーレート、TOPIXベータ、PBR1倍割れ要請への考慮など）を定義したスキル。
- **`earnings-analysis`**: 決算リリースやコールの文字起こしから、定性的・定量的な変化や経営陣のトーンを抽出。
- **`initiating-coverage`**: 新規カバレッジ開始レポートを作成するための5フェーズ（企業リサーチ、モデリング、バリュエーション、チャート生成、レポート組み立て）のガイドライン。
- **`audit-xls`**: スプレッドシートの整合性チェック、循環参照の監査、数式のエラー検出を行う。
- **`xlsx-author` / `pptx-author`**: 成果物としての Excel（.xlsx）や PowerPoint（.pptx）ファイルの headless 生成・書式適用ルール。
- **`morning-note` / `thesis-tracker` / `catalyst-calendar`**: レポート、投資仮説、イベントカレンダーの管理手順。
