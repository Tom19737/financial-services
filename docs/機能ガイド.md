# 機能ガイド：コマンド・エージェント・スキル早見表

本プロジェクトに導入した [Claude for Financial Services](https://github.com/anthropics/financial-services) の各機能を「実行すると何ができるか」の観点で解説する。配布元の定義ファイル(commands/*.md, agents/*.md)の実体に基づく。

機能は3層に分かれる。

- **スラッシュコマンド** … `/プラグイン名:コマンド名` で**明示的に**起動する入口。多くは対応するスキルを読み込んで実行する薄いトリガー。
- **エージェント** … 複数スキルを順に呼び、ワークフローを**端から端まで**回す。成果物ごとに停止して人間のレビューを挟む。
- **スキル** … コマンド/エージェントから呼ばれる実作業の本体。文脈一致時に自動発火することもある。

> 注意：いずれも**草案(ドラフト)を作る**ツールであり、投資助言・推奨や取引執行は行わない。数値は出典明記が原則で、出典不明値は `[UNSOURCED]` と明示される設計。最終的な検証と承認は利用者の責任。

## 1. equity-research（エクイティリサーチ・バーティカル）

カバレッジ銘柄に対する一連のリサーチ業務をコマンド化したもの。

| コマンド | 引数 | 実行すると何ができるか | 呼び出すスキル |
|---|---|---|---|
| `/equity-research:earnings` | 銘柄, 四半期 | 四半期決算を分析し、**8〜12ページのDOCX決算アップデートレポート**を生成。コンセンサス対比のbeat/miss定量化、セグメント/マージン分析、ガイダンス変化、投資仮説への影響、8〜12点のチャート、出典リンク付き。実行前に最新データ(直近3ヶ月以内)であることを検証する | earnings-analysis |
| `/equity-research:earnings-preview` | 銘柄 | **決算発表前**のプレビュー。コンセンサス予想、注目すべきKPI、強気/標準/弱気のシナリオを整理 | earnings-preview |
| `/equity-research:initiate` | 銘柄 | **新規カバレッジ開始レポート**を5タスクのワークフローで作成(機関投資家品質のイニシエーション。最も重いワークフローで on-invoke 約11.7k tok) | initiating-coverage |
| `/equity-research:model-update` | 銘柄 | 既存の財務モデルに新しい決算・ガイダンス・改訂前提を反映 | model-update |
| `/equity-research:morning-note` | なし | 担当ユニバース横断で、**朝会用の簡潔なノート**を作成。一晩の動き、決算反応、トレードアイデアを集約 | morning-note |
| `/equity-research:screen` | スクリーニング条件 | 定量スクリーン/テーマ的スイープで**新規投資アイデアを抽出**。条件未指定ならロング/ショート・セクター・スタイル・テーマを質問 | idea-generation |
| `/equity-research:sector` | セクター/業界 | **セクター概観レポート**。市場規模、競争環境、投資含意をカバー | sector-overview |
| `/equity-research:thesis` | 銘柄 | **投資仮説(thesis)の新規作成・更新**。新しいデータポイントを反映して仮説を管理 | thesis-tracker |
| `/equity-research:catalysts` | 期間(既定:今後2週間) | カバレッジ横断の**カタリスト・カレンダー**を構築・確認 | catalyst-calendar |

## 2. financial-analysis（コア・モデリング）

DCF/comps/LBO等の汎用的な財務モデリングとExcel/デッキ作業。

| コマンド | 引数 | 実行すると何ができるか | 呼び出すスキル |
|---|---|---|---|
| `/financial-analysis:comps` | 銘柄 | **類似企業比較分析(trading comps)**を機関投資家品質で構築。4〜6社のピア選定→営業指標(売上/成長率/粗利/EBITDA)とバリュエーション倍率(EV/Rev, EV/EBITDA, P/E)→統計サマリ(最大/75%/中央値/25%/最小)を**数式ベースのExcel**で出力。入力=青/数式=黒、ハードコード値はセルコメントに出典 | comps-analysis |
| `/financial-analysis:dcf` | 銘柄 | **DCFバリュエーションモデル**を構築。まずcompsを回してピア倍率を取得→それをターミナル倍率や前提のベンチマークに利用→Bear/Base/Bullシナリオ、WACC(CAPM)、感応度表→株価インプライ。最後にDCFインプライド倍率をピア中央値とクロスチェック。出力はcomps と DCF の2つの.xlsx | comps-analysis → dcf-model |
| `/financial-analysis:lbo` | 企業/ディール条件 | **LBOモデル**(レバレッジドバイアウト)を構築。Sources&Uses、デットスケジュール、リターンウォーターフォール、IRR/MOIC感応度 | lbo-model |
| `/financial-analysis:3-statement-model` | テンプレートのパス | **3表連動モデル**(PL/BS/CF)をテンプレートに沿って作成 | 3-statement-model |
| `/financial-analysis:competitive-analysis` | 企業/業界 | **競合環境分析**。プレイヤー、ポジショニング、競争基盤を整理 | competitive-analysis |
| `/financial-analysis:debug-model` | .xlsxのパス | 既存モデルを**監査・デバッグ**。壊れた数式、BS不一致、ハードコード上書き、循環参照、ロジックエラーを検出(BS balance/キャッシュtie-out/roll-forward等のモデル整合性チェック含む) | audit-xls (scope=model) |
| `/financial-analysis:ppt-template` | .pptx/.potxのパス | 自社のPowerPointテンプレートから、**ブランド準拠のデッキ生成スキル**を自動生成(レイアウト/プレースホルダ解析→専用スキル化)。「自社の様式」をClaudeに教え込む入口 | ppt-template-creator |

## 3. エージェント（ワークフロー一括実行）

サブエージェントとして起動。複数スキルを順に呼び、各成果物でレビュー停止する。

| エージェント | トリガー | 実行すると何ができるか | 主な出力 |
|---|---|---|---|
| **market-researcher** | セクター/テーマの調査依頼(単一銘柄ではない) | セクターまたはテーマの**一次調査レポート**を作成。①業界概観(規模/成長/構造/ドライバー/why-now)→②競合環境→③ピアcompsスプレッド→④テーマを表す3〜5銘柄のアイデアショートリスト→⑤リサーチノート(任意でスライド)。comps後とノート後の2回、レビューのため停止 | リサーチノート(+任意のpptx) |
| **earnings-reviewer** | カバレッジ銘柄が決算発表したとき | **決算イベントを端から端まで**処理。①実績/コンセンサス/10-Q・8-K取得+コール全文読込(要約ではなく原文)→②コール分析(ガイダンス/トーン/経営陣が避けた論点)→③カバレッジモデル更新(全変更セルに出典)→④モデルQC(audit-xls)→⑤ノート草案。外部公開は行わない | 更新済みモデル / 決算ノート草案 / 差異テーブル(実績vsコンセンサスvs従来予想) |
| **model-builder** | ゼロからの新規モデル作成(既存更新ではない) | **完全リンクのExcelモデルを新規構築**。DCF/LBO/3表/compsのいずれか。①CapIQ/Daloopaで実績取得→②該当スキルで構築(青/黒/緑の色分け、計算セルにハードコード禁止)→③audit-xls監査→④感応度表→⑤レビュー停止。全出力は数式、ハードコード前提は出典明記か`[ASSUMPTION]`表示 | 監査済みの.xlsxモデル |

> エージェントやスキルは `yfinance` による自動取得を最優先します。`yfinance` が失敗した場合、または明示的な指示がある場合は、GASを利用したGoogleスプレッドシート（Google Financeから蓄積したデータ等）からのデータ取得を実行します（詳細は日本株ローカライズ共通の無料データパイプラインを参照）。有料のMCPデータコネクタや手動Web検索は、これら自動パイプラインが動作しない場合のフォールバックとして位置づけられます。

## 4. スキル（自動発火する実作業の本体）

コマンド/エージェント経由のほか、関連作業の文脈で自動的に参照される。代表例：

| スキル | 役割 |
|---|---|
| `dcf-model` / `lbo-model` / `3-statement-model` / `comps-analysis` | 各バリュエーションモデルの構築本体 |
| `earnings-analysis` | 決算リリース・10-Q・コール文字起こしからの分析 |
| `initiating-coverage` | カバレッジ開始レポートの5タスク・ワークフロー本体(最も大きい) |
| `sector-overview` / `competitive-analysis` | 業界・競合分析 |
| `idea-generation` | スクリーニング/アイデア創出 |
| `audit-xls` | Excelモデルの監査(整合性・数式・ハードコード検査) |
| `xlsx-author` / `pptx-author` | Excel・PowerPoint成果物の生成 |
| `morning-note` / `thesis-tracker` / `catalyst-calendar` | 朝会ノート・仮説管理・カタリスト管理 |

---

## 5. 典型的な組み合わせ方

- **新規銘柄を一から調べる**：`/equity-research:sector`(業界把握) → `/equity-research:screen`(候補抽出) → `/equity-research:initiate`(カバレッジ開始) → `model-builder`(バリュエーションモデル) → `/equity-research:thesis`(仮説確定)
- **決算が出た**：`earnings-reviewer` 一発(モデル更新+ノート+差異表)、または手動で `/equity-research:earnings`
- **バリュエーションだけ素早く**：`/financial-analysis:comps` → `/financial-analysis:dcf`
- **テーマ投資の調査**：`market-researcher`(セクター/テーマ → 概観+競合+comps+アイデア)
- **既存モデルの健全性チェック**：`/financial-analysis:debug-model`

## 補足：使い始め方

1. プラグインは**次回セッション開始時**に有効化される(本ガイド生成時点では適用待ちの可能性あり)。
2. コマンド一覧はセッション内で `/` を入力して確認できる。
3. 多くのコマンドは引数省略時に対話的に質問する(銘柄・四半期・条件など)。
4. 元スキルは米国式の開示・comps定義が前提だが、**日本株向けローカライズ対応済み**。共通の読み替えは [tools/localization/japan-equity-overrides.md](../tools/localization/japan-equity-overrides.md)、各スキル固有の上書きは対象 SKILL.md 末尾の「日本株対応」節に追記済み(有報/決算短信/TDnet・EDINET、円/百万円、3月期決算、証券コード4桁、会社予想＋コンセンサス、PBR/ROE重視、DCFは10年JGB・TOPIXベータ・円ベース 等)。
