# ワークフロー: /save-session

手動で現在の会話コンテキストまたはトランスクリプト（会話履歴）をジャーナルに保存します。

## オプション引数
- `--tags "タグ1,タグ2"`: セッションに付与する任意のタグを指定します（例: `NAND,設計`）。
- `--notes "追加メモ"`: セッション要約の「結論」または「追加メモ」セクションに含めるメモを指定します。

## 実行手順

1. **引数の解析**
   - ユーザーが指定した `--tags` と `--notes` を抽出します。指定されたタグは、コンテキストから自動抽出されたタグに加えて `tags` 配列に追加されます。指定されたメモはセッション要約に追加されます。

2. **会話履歴の確保とフォールバック**
   - 会話コンテキストから直近の目的、判断事項、結論を抽出します。
   - 会話が非常に長く、コンテキストウィンドウの上限により過去のやり取りが一部消失していることが疑われる場合は、以下のパスからトランスクリプト（会話履歴）のJSONLファイルを読み込み、消えてしまった部分の判断ログや要約を補完します。
     - トランスクリプトファイルの場所: `C:\Users\fwhrv\.gemini\antigravity\brain\[Conversation_ID]\.system_generated\logs\transcript.jsonl`
       （※ `[Conversation_ID]` は現在のセッションIDに置き換えます。ファイルが存在しないか読み込めない場合は、現在コンテキストにある情報のみで要約を行います）

3. **ジャーナルの保存実行**
   - [journal-save スキル](file:///d:/Programming/Application/financial-services/.agents/skills/journal-save/SKILL.md) をロードして実行します。
   - `trigger` フィールドには `"manual"` を設定します。
   - `workflow` フィールドには、直前に行っていた作業のワークフロー名（`dcf`, `comps`など）があればそれを指定し、単なる壁打ちや設計議論の場合は `"freeform"` を設定します。

4. **結果の報告**
   - 正常に保存された場合、ユーザーに「セッション [Session_ID] を保存しました」と、作成されたファイルパスを提示して完了を報告します。
