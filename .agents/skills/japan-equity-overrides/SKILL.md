# 日本株ローカライズ共通オーバーライド

`tools/` 配下のスキル/コマンド/エージェントは米国式（英語開示・USD・SEC EDGAR・US GAAP）前提で書かれている。**日本株を対象とする場合、以下の読み替えを常に適用する。** 各スキル末尾の「日本株対応」節はこの共通ルールを前提に、スキル固有の上書きのみを記載する。

## 1. 開示制度・一次情報

| 米国（原文） | 日本（読み替え） | 入手元 |
|---|---|---|
| 10-K（年次） | 有価証券報告書（有報） | EDINET（金融庁） |
| 10-Q（四半期） | 四半期報告書は2024年廃止 → **決算短信（四半期）** で代替、半期報告書 | TDnet（JPX）／EDINET |
| 8-K（適時開示） | 適時開示（決算短信・業績予想修正・TOB等） | TDnet（JPX） |
| Earnings release | 決算短信（サマリー＋添付）／決算説明会資料 | TDnet／会社IR |
| DEF 14A（委任状） | 招集通知／株主総会参考書類 | 会社IR／TDnet |
| S-1（登録届出） | 有価証券届出書（IPO） | EDINET |
| 10-K/10-Q 注記 | 有報の経理の状況・注記 | EDINET |

- **一次情報の優先順**: ①EDINET（有報・半期・大量保有報告書）②TDnet（適時開示・決算短信）③統合報告書／決算説明会資料 ④会社四季報（東洋経済）。
- 原文に「SEC EDGAR」「10-K」「10-Q」「8-K」とあれば上表で読み替える。

## 2. 会計基準

- 日本企業は **日本基準（J-GAAP）／IFRS／米国基準** が混在。銘柄ごとに採用基準を確認し明示する。
- **経常利益**は日本基準特有（営業利益＋営業外損益）。IFRS/米国基準には無い。beat/miss やモデルでは営業利益・経常利益・親会社株主に帰属する当期純利益を区別する。
- **のれん**: J-GAAPは規則償却、IFRSは非償却（減損のみ）。ピア比較時は基準差を注記。
- 「Operating Income」「Net Income」は「営業利益」「親会社株主に帰属する当期純利益」に対応。

## 3. 通貨・単位

- 通貨は **円（JPY, ¥）**。財務数値は **百万円** 単位が標準（大企業は十億円も可）。原文の「USD Millions」は「百万円」に置換。
- 海外売上・海外子会社は為替前提（USD/JPY 等）を明示して換算。

## 4. 決算期・カレンダー

- 多くの日本企業は **3月期決算（FY末 = 3/31）**。四半期は Q1=4–6月, Q2=7–9月, Q3=10–12月, Q4=1–3月。ただし12月期・2月期等もあるため銘柄ごとに確認。
- 本決算発表は5月、中間（上期）は11月が多い。**上期/下期（中間決算）**の概念を用いる。
- 業績予想は **会社自身が通期予想（会社予想）を開示**するのが日本の特徴。修正時は「業績予想の修正」を適時開示。

## 5. 識別子・市場区分

- **証券コード = 4桁**（例: トヨタ自動車 7203、ソニーG 6758）。米国ティッカー表記の箇所はコードに置換。
- 市場区分（2022年再編）: **プライム／スタンダード／グロース**。旧「東証一部」等の表記は新区分に。
- 指数: **TOPIX**（時価総額加重）、**日経平均**（株価平均型）、TOPIX-17/東証33業種（業種分類）。

## 6. データソース・コンセンサス

- コンセンサス: **QUICKコンセンサス／IFIS／東洋経済予想**。加えて Bloomberg／FactSet／LSEG(Refinitiv)。
- **会社予想 vs アナリストコンセンサス**の両方を対比する（日本は会社予想が基準になりやすい）。
- 「Bloomberg」「FactSet」を一次情報のように扱う原文は、日本では EDINET/TDnet（一次）＋QUICK/四季報（補助）を優先。

## 7. バリュエーション慣行

- 倍率は PER・PBR・EV/EBITDA・配当利回りが中心。米国より **PBR を重視**（東証の「資本コストや株価を意識した経営」＝PBR1倍割れ改善要請、2023年〜）。
- **ROE**（伊藤レポートの8%目安）、**配当性向・総還元性向**（自社株買い含む）を重視。
- **政策保有株（持ち合い）**の縮減、**親子上場・少数株主**の利益相反は日本固有の論点。EV計算・ガバナンス評価で考慮。
- 実効税率は **約30〜31%**（モデルの税率デフォルト）。

## 8. DCF前提（円ベース）

- リスクフリーレート = **10年国債（JGB）利回り**。
- ベータは **TOPIX 対比**で推計。エクイティ・リスクプレミアムは日本株の水準。
- WACC・FCF・割引は **円ベース**で統一。海外事業は為替前提を明示。

## 9. カタリスト・イベント（日本固有）

- 自社株買い・増配・累進配当、**政策保有株の縮減**、**東証PBR要請への対応**、TOB／MBO（二段階買収・スクイーズアウト）、アクティビスト、ガバナンス改善。
- **6月の株主総会集中**、3月末・9月末の配当／株主優待の権利確定、業績予想の修正開示。

## 10. その他

- 朝会ノートは前日の **米国市場引け・USD/JPY・ADR・SQ** 等を織り込み、寄り付き前（8時台）を想定。
- 市場・シェア統計は官公庁・業界団体（経産省、矢野経済研究所、富士経済 等）。
- 株主優待・累進配当など、日本特有の株主還元慣行に留意。

## 11. 無料データ取得パイプライン（yfinance / GAS連携）

有料データコネクタ（FactSet/Morningstar等）が利用できない環境では、以下のローカルスクリプトを利用してデータを取得する。

### A. 株価情報・基本指標の自動取得
- スクリプト: [fetch_yfinance.py](file:///d:/Programming/Application/financial-services/scripts/fetch_yfinance.py)
- 実行方法:
  ```bash
  .\.venv\Scripts\python scripts/fetch_yfinance.py <ティッカーまたは証券コード4桁>
  ```
- 動作:
  - 証券コード4桁（例: `7203`）が渡された場合、自動的に `.T` を付加して東証からデータを取得する。
  - 出力先: `./out/market_data/<ティッカー>.csv`（ヒストリカル価格）および `./out/market_data/<ティッカー>.json`（現在株価、時価総額、発行済株式数等）。
  - 財務モデル構築の際は、上記JSONファイルから最新データを読み込み、ExcelのInputsシートにインポートして適用する。

### B. Googleスプレッドシート（Google Finance蓄積データ含む）の取得
- スクリプト: [fetch_gas_sheets.py](file:///d:/Programming/Application/financial-services/scripts/fetch_gas_sheets.py)
- 実行方法:
  ```bash
  .\.venv\Scripts\python scripts/fetch_gas_sheets.py <GASウェブアプリのデプロイURL>
  ```
- 動作:
  - スプレッドシート側のGASウェブアプリからJSON形式でデータを取得。
  - 出力先: `./out/sheet_data.json`。
  - スプレッドシート側のGASのソーステンプレートは [gas_template.js](file:///d:/Programming/Application/financial-services/scripts/gas_template.js) を参照のこと。
